Every code generation task that modifies existing executable application code MUST execute a single benchmarking pass on the target module's public entry points BEFORE applying changes, measuring execution time and memory allocation, and MUST execute an identical pass AFTER changes, producing a comparison report with per-metric deltas. Modified code MUST NOT degrade any measured metric by more than 10% without an explicit justification annotation in the comparison report. If the existing code cannot be executed for benchmarking (no runnable entry points, external dependency unavailable), the agent MUST document the reason and deliver an after-only measurement. Verification: every delivery that modifies existing code includes a before/after comparison report or a documented exemption with after-only measurements; any delivery missing both fails review. Scope: all Blitzy code generation agent outputs that modify existing executable application code as part of general code review; greenfield module creation requires after-only measurement; excludes configuration files, documentation, and static assets.
